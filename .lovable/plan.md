

## Проблемы и план исправления

### Проблема 1: Кнопка "Выйти" не работает

**Причина**: В `onAuthStateChange` используется `async/await` — при вызове `signOut()` колбэк пытается выполнить запрос к базе (`await supabase.from('user_profiles')...`), что создает блокировку (deadlock). Supabase не может обработать следующее событие, пока предыдущий `await` не завершится.

**Решение**: Убрать `await` из колбэка `onAuthStateChange`. Для события `SIGNED_OUT` сразу очищать состояние без запросов к БД. Для `SIGNED_IN` / `TOKEN_REFRESHED` — обновлять профиль через "fire and forget" или вынести логику наружу.

### Проблема 2: Пользователи не загружаются в админ-панели

**Причина**: RLS-политика `Admin can view all profiles` проверяет email в JWT. Нужно убедиться, что сессия полностью восстановлена до выполнения запроса. Также возможна гонка состояний — компонент `AdminPanel` монтируется и делает запрос до того, как `auth.getSession()` завершится.

**Решение**: Добавить проверку готовности сессии перед загрузкой данных в `AdminPanel`.

### Изменения в файлах

**1. `src/hooks/useAuth.ts`**
- Переписать `onAuthStateChange`: не использовать `await` внутри колбэка
- Для `SIGNED_OUT` — мгновенно очищать `user` и `localStorage`
- Для `SIGNED_IN` / `TOKEN_REFRESHED` — загружать профиль отдельной функцией (не блокируя колбэк)
- Функцию `logout` сделать надежнее: сначала очистить UI, потом вызвать `signOut`

**2. `src/components/AdminPanel.tsx`**
- Перед загрузкой списка проверять, что сессия активна (`getSession()`)
- Если сессия не готова, ждать с индикатором загрузки
- Добавить лучшую обработку ошибок для диагностики
